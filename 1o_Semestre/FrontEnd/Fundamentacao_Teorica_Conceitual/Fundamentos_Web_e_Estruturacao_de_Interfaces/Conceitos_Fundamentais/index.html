A tecnologia por trás da comunicação Web: Http e Https

HTTP, do inglês Hypertext Transfer Protocol ou Protocolo de Transferência de Hipertexto, é um protocolo (conjunto de regras que padronizam como dois dispositivos finais se comunicam em uma rede) utilizado na comunicação
web. Quando usamos um navegador em nossa máquina, por exemplo, estamos utilizando um Software da camada de aplicação destinado a estas comunicações web. Ao usarmos o cliente Browser e digitarmos um link na
barra de pesquisa, quando estabelecemos comunicação e visualizamos o respectivo site e o arquivo retornado, na verdade o que ocorreu foi um (ou mais) processo(s) HTTP de requisição e resposta. Já o HTTPS significa
HTTP Secure, e aplica tecnologias de criptografia TLS / SSL para garantir segurança na comunicação. Sendo uma condição quase obrigatória para qualquer projeto que prese por Integridade e Confiabilidade de dados
em suas soluções, o HTTPS e as Certificações SSL/TLS são triviais para manter a segurança e confiabilidade no serviço.

Estrutura de Pastas em um Projeto Front-End

A estruturação de pastas e arquivos em um projeto Front-End é um fator muito importante para garantir legibilidade, organização e que o projeto não acabe ficando confuso ao passar do tempo.
Sabemos que qualquer tarefa que façamos que exija lapidação, com o tempo tendemos a tornar o produto final de acordo com o que imaginamos ser a melhor vesão deste produto. Organizar
corretamente nosso ambiente de trabalho significa não nos perdermos no futuro e garantir uma boa leitura e praticidade das tarefas. Em geral, uma estrutura de pastas é geralmente semelhante a:

  • Pasta do Projeto
    • index.html (Arquivo Principal (home) do projeto)
    • assets/
      • fonts/
        • * Arquivos de Fontes (.ttf, etc)
      • icons/
        • Arquivos de Ícones
    • css/
      • Arquivos de estilização (.css)
    • docs/
      • README.md (exemplo)
    • img/
      • Arquivos de imagem ou multimídia
    • js/
      • Scripts (neste caso javascript)

ps: Este repositório está utilizando esta estrutura como exemplificação.


Registro de Domínio

O domínio é o endereço multicaracteres único que identifica uma máquina na internet. Diferente do endereço lógico Ipv4 ou Ipv6 que efetivamente identifica uma máquina em uma rede, o domínio é um objeto em texto, que é convertido em endereço lógico através do protocolo DNS (Domain Name System). Este protocolo foi criado pensando em facilitar o acesso a servidores e máquinas, tendo em vista que digitar manualmente cada dígito de um endereço IP seria inviável do ponto de vista prático.

Propagação DNS

Um domínio pode levar cerca de até 48 horas para se propagar totalmente. Durante este período, dependendo do SIP e da localização do usuário, este pode não conseguir acessar o destino. Por este motivo é conveniente realizar mudanças DNS com aviso prévio e fora de horários de pico de acesso.

HTML5: A linguagem de Marcação da Web Moderna

HTML5 (HyperText Markup Language versão 5) representa a evolução do HTML antigo para uma melhor versão com Semântica HTML, API's Javascript avançadas, melhor suporte multimídia, dentre outras vantagens.
A declaração: 

<!DOCTYPE HTML5>
É o primeiro elemento de qualquer documento HTML5, informando ao navegador que os documentos referentes seguem o padrão desta versão. Abaixo um exemplo de um código HTML5 básico: 
       

<!DOCTYPE HTML>
<HTML LANG="pt-BR">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Título</title>
</head>
<body>
                    ...
</body>
</html>

Anatomia das Tags HTML

Existem dois tipos de Tags HTML: As que possuem abertura e fechamento e as que possuem autofechamento. A anatomia de uma tag HTML de abertura e fechamento é: <tag>(conteúdo)</tag>,
enquanto a anatomia de uma tag de autofechamento seria <tag/>. As tags funcionam como containers, o conteúdo que está dentro de uma tag é influenciado de acordo com a função e configuração
desta tag em seu código.
